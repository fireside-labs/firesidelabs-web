The Letter-Shuffle Contamination Probe

The Letter-Shuffle Contamination Probe

A Practical Technique for Detecting Benchmark Memorization

Multiple-choice benchmarks dominate LLM evaluation. MMLU, 500Q, hundreds of standardized tests all rely on choosing between lettered options (A, B, C, D). But here's the problem: models can score high on these benchmarks without actually reasoning. They can memorize that certain letters appear more frequently in certain positions.

A model scoring 88% on MMLU via memorization is useless in production. It won't generalize to new questions, paraphrased questions, or questions where letter order matters.

This paper introduces the letter-shuffle contamination probe — a simple, cheap technique to detect memorization. The probe works by running the benchmark normally, then re-running with letters shuffled (W/X/Y/Z instead of A/B/C/D). Models with real reasoning drop ≤2pt. Models with memorization drop 5-15pt.

We validate the probe across 25 production models. One "successful" model optimization was revealed as contamination-driven, not real improvement.

The Problem: Invisible Memorization

When you benchmark a model on MMLU, you're asking it to choose the correct answer among four options. The implicit assumption: the model is reasoning and selecting based on that reasoning. But models don't always reason. Sometimes they pattern-match.

A model that has seen enough MMLU-like text during training can learn letter distributions, positional biases, and stylistic cues — none of which requires understanding the actual question.

Why Existing Methods Fail

Held-out test sets catch explicit overlaps but miss structural similarities. Contamination audits grep training data but miss pattern-matching. Performance caps don't work because training dates are fuzzy and models learn general patterns. The letter-shuffle probe is different: it's cheap, sensitive, and catches what other metrics miss.

The Methodology

Step 1: Run the benchmark normally with standard A/B/C/D options.

Step 2: Re-run with letters shuffled (W/X/Y/Z), gold answers remapped accordingly.

Step 3: Compare accuracy. Models with real reasoning drop ≤2pt. Models with memorization drop 5-15pt.

Why does letter position matter? Language models process tokens sequentially. The token sequence "(A)" appears in the same position in every question. A model with memorization will prefer "(A)" when it appears early, and fail to redirect when letters change. A model with real reasoning will read the option text and select based on correctness.

The Specialist Variant A Discovery

This model appeared to match baseline performance (Qwen3-14B) on standard benchmarks at 88.3%. But under letter-shuffle, it dropped to 80.6% while baseline dropped only 4.8 points. The "tie" was contamination, not real reasoning. The model was 3.8pt worse in actual capability.

18 of 25 models show no concerning memorization. 7 of 25 show heavy contamination that should disqualify deployment.

Practical Recommendations

1. Make the Probe Mandatory: Build it into every benchmark pipeline as a validation pass. It costs nothing and catches failures invisible to other metrics.

2. Report Both Numbers: Every benchmark result should include standard accuracy AND shuffled accuracy, with delta reported explicitly.

3. Flag High-Δ Models: Models showing ≥5pt sensitivity should be flagged in results with explicit contamination warnings.

4. Extend to Other Benchmarks: The letter-shuffle technique works for any multiple-choice benchmark. For code (variable-name shuffle), math (number-shuffle), apply the same principle.
